HTML

  • Tag: Also known as opening and closing tags, most often represented by angle brackets with associated tag names contained inside. Closing tags begin with a forward slash

  • Body Element: Defines the body of the document, which contains the content that is visible to the user. Body Elements are also examples of Semantic HTML elements.

  • Child Elements: Are elements nested inside other elements.
  • Semantic HTML Elements: Name of the element describes its content and the role it plays on the webpage. Makes YOUR HTML easier to read by other developers and the browser. Semantic HTML elements also play an important role in making sure your webpage is accessible to the widest variaty of users.

  • Decleration: Tells the browser what document type it should expect when it is opened. Declerations are not case sensitive as HTML elements are.

  • Metadata: Data about our webpage that the browser uses to effectively serve our webpage to users. Contained in the header not typically seen by users, but contains important information about accessibility, search engines, and performance.

  • SEO: Search Engine Optimization

  • head Contains information, or metadata, that helps the browser render the page correctly. Elements contained within the head are not visible when a user looks at a webpage.

  • body Contains the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.

  • title Is required and is extremely important for SEO.

  • base Defines the base URL for a webpage.

  • link Connects an external resource to the HTML document.

  • meta Defines metadata such as the character set, description, keywords, author, and viewport.

  • script Embeds client-side scripts in the HTML document.

  • div Creates a division in the page.

  • span Creates an inline division.

  • p Paragraph or block of text.

  • ul Unordered list or bullet points.

  • ol Ordered list

  • li List items

  • img Contains information about images displayed on a webpage

  • h1-6 Represents level of heading in a given text block. 1 being largest text size and 6 being the smallest.

  • br / Creates a line of empty space

  • a Stands for anchor, creates links to the same page or other webpages.

  • href Points to the url for the link

  • src This is the most important image attribute because it defines the source or location of the image file

  • alt Contains the text string that describes the appearance and functionality of the image
  • HTML attributes are always defined in the beginning element, never the closing element

CSS Section

  • A margin indicates how much space we want around the outside of an element

  • A padding indicates how much space we want around the content inside an element.

  • Inline CSS is used when a develpoer want to style an element directly in the HTML file. Applies to a specific HTML tag using a style attribute with a CSS rule to style a particular page element only. Useful for quick and permanent changes but are less flexible than internal and external style sheets, because each inline style must be edited induvidually whenever a design change is made.

  • Seperation of Concerns is a widely used term that refers to seperating code into diffrent files based on use.

  • A decleration contains two important components ie the CSS property we want to apply and the value.

  • Every HTML element is essentially a box. Each box consist of margins outside the element, the element borders, padding, and content referred to ast the CSS Box Method.

  • link Creates the connection to the external file.

  • rel Specifies the relationship between current and linked document

  • href Specifies location (URL) of the external source using a relative path

  • .card Period before card designates this a class selector, any HTML element assigned to the class (card) will gain these property assignments form the CSS.

  • box-shadow Adds shadow effects around an elements frame, multiple effects can be set using a comma.

GIT Section

  • Always make sure you are in the correct directory.

  • Commit often.

  • Do Not Forget to close parentheses!

  • Do Not Push until you are ready!

  • Pull often!

  • pwd Print Working Directory, shows what you are currently in.

  • clear Clears window.

  • cd "name of directory" Change directory, moves you into directory.

  • cd .. Moves you back one directory.

  • cd ~ Takes you all the way back to home directory.

  • tab Completes directory name that you started typing.

  • ls Shows a list of all files in current directory.

  • mkdir Make directory.

  • touch Creates a new file.

  • git add . & add -A Adds and keeps track of files in local git repository stagging area.

  • git init Initializes or creates a local repository to store commits.

  • git commit -m"message about changes being commited" Command for saving progress like a checkpoint.

  • git status Shows everything in stagging area.

  • git checkout "name of branch" switches to named branch.

  • git clone Creates a copy of a repository in a new directory.

  • git pull Updates the local directory from a remote directory.

  • git push Pushes a copy of the local directory to a remote directory.

JAVASCRIPT Section

  • A variable is a named container that allows us to store data in our code. Variables can be declared with the keyword let, followed by the name you give to the variable.

  • Variables are necessary to do anything interesting in programming. If values could not change than nothing dynamic could be done.
  • Control flow is the order in which a computer executes code in a script. In JavaScript code runs from the first statement at the top of the page, down to the last statement at the bottom of the page.

  • Array is a single variable that is used to hold a group of data. 0 indexed meaning first item in array has an index of 0, second 1, third 2 etc.
  • Array stucture that allows you to store multiple values in a single reference ie let myVariable = [1, 'Bob', 'Steve', 10];

  • semi colon at the end declares the end of the statement.

  • Boolean= true false value. The words true and false are keywords no qoutation needed ie let myVariable = true;

  • Number signifies this is a number with no qoutes ie let my Variable = 10;

  • String is a swquence of text sgnified by single qoute marks ie let myVariable = 'Bob';

  • Object can be anyghing. Everything in JavaScript is an object and can be stored in a variable.

  • if statement is a way to introduce decision making into our code.

  • === Triple equal operator checks to see if two values are equal and returns a Boolean result true if the values are equal and false if values are not.

  • To declare a function use keyword function, give the function a unique name followed by parentheses then add curly brackets that contain the code for the task we want to perform.
